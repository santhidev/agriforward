# Page Routing & UI Flow

> เอกสารนี้บันทึกโครงสร้างหน้า (page structure) และ flow หลักของ AgriForward MVP โดยอิงจาก Prototype ที่ปรับปรุงแล้ว (22 ไฟล์ HTML + CSS) และ PRD-MVP-v3 + CONTEXT.md สำหรับการพัฒนา Next.js App Router

> **Prototype:** ดู prototype/README.md สำหรับ flat static HTML (23 หน้า) ใช้ทดสอบ UX กับเกษตรกร

---

## หลักการออกแบบ

- **Dual-Role Account**: ผู้ใช้รายเดียวทำได้ทั้งซื้อและขาย ไม่แบ่ง Role ในชื่อ URL (`/buyer/*`, `/seller/*` ไม่มี)
- **Entry Point หลัก**: `/products` — หน้าเดียวที่มีทั้งซื้อและขาย พร้อมสินค้าติดตามด้านล่าง
- **Farmer-First Language**: ไม่ใช้ภาษาราชการ/เทคนิคบนหน้า UI — ไม่มี "PO" "Offer" "Escrow" ผู้ใช้เห็นแค่ "งาน" "ขาย" "ซื้อ" "เงิน"
- **Nav เหลือ 4 อัน**: 🌿 สินค้า | 📋 งานของฉัน | 🔔 ข่าวสาร | 👤 ฉัน

## 1. Public & Auth

| หน้า | Route | คำอธิบาย |
|------|-------|----------|
| **Landing** | `/` | แนะนำแพลตฟอร์ม, CTA ไปดูสินค้า/สมัคร |
| **Login** | `/auth/login` | เข้าสู่ระบบ |
| **Register + OTP** | `/auth/register` | สมัครด้วยเบอร์โทร + OTP |
| **Onboarding** | `/auth/onboarding` | กรอกข้อมูลครั้งแรก (ฟาร์ม/บริษัท, ที่อยู่) |
| **PDPA Consent** | `/auth/consent` | ยินยอม PDPA (ครั้งเดียว, ไม่ย้อนกลับ) |

## 2. Product Catalog (Entry Point หลัก)

| หน้า | Route | คำอธิบาย |
|------|-------|----------|
| **Product List** | `/products` | รายการสินค้าทั้งหมด — หน้าแรกหลัง Login แสดงราคาล่าสุด (ตัวใหญ่) + ปุ่ม "ซื้อ" (สีฟ้า) / "ขาย" (สีเขียว) ต่อสินค้า ด้านล่างแสดงสินค้าที่ติดตาม (แท็บ ทั้งหมด/ติดตามแล้ว) พร้อม badge จำนวนงานที่เปิดรับ |
| **Product Detail** | `/products/[id]` | รายละเอียดสินค้า + ราคาตลาดล่าสุด + จำนวนคนซื้อ/ขาย + ปุ่ม "ฉันอยากซื้อ" (สีฟ้า) → ไปหน้าสร้างงานซื้อ + Section "มีคนซื้ออะไรบ้าง" → แสดงงานซื้อที่เปิดรับ (PO cards) พร้อมปุ่ม "ฉันจะขาย" → ไปหน้ายื่นขาย |

> **หมายเหตุ:** Watchlist รวมอยู่ใน `/products` แล้ว (แท็บ "ติดตามแล้ว") ไม่มีหน้าแยก

---

## 3. งานซื้อ (Buyer Flow)

| หน้า | Route | คำอธิบาย |
|------|-------|----------|
| **Create PO** | `/orders/new?productId=...` | "อยากซื้ออะไร" — แค่ 2 คำถาม: (1) ต้องการกี่กิโล? (2) อยากได้ราคาเท่าไร? ไม่ต้องเลือกวิธีรับสินค้า (เลือกตอนจ่ายเงิน) |
| **My Jobs** | `/orders` | "งานของฉัน" — แท็บ 2 แท็บ: "กำลังซื้อ" (PO cards) / "กำลังขาย" (offer cards) — ครอบคลุมทั้งซื้อและขายในหน้าเดียว |
| **Job Detail** | `/orders/[id]` | "รายละเอียดงาน" — ข้อมูลง่าย + คนที่มาขาย (offers) + คุยกัน (comment thread) |
| **Payment** | `/orders/[id]/checkout` | "จ่ายเงิน" — สรุปราคาใหญ่ + เลือกวิธีรับ: 2 ปุ่มใหญ่ (รับเองฟรี / ส่งถึงบ้าน +3,000) |

## 4. งานขาย (Seller Flow)

| หน้า | Route | คำอธิบาย |
|------|-------|----------|
| **Submit Offer** | `/offers/new?poId=...` | "จะขายเท่าไร" — แค่ใส่ราคา + คำนวณ real-time: "ได้เงินรวม: X บาท" + "จ่ายมัดจำ: X บาท (คืนเมื่อขายสำเร็จ)" |
| **Offer Detail** | `/offers/[id]` | "งานขาย" — สถานะง่าย (ถอน/ตอบรับ/ปฏิเสธ) |

> **หมายเหตุ:** Marketplace รวมอยู่ใน `/products/[id]` แล้ว (Section "มีคนซื้ออะไรบ้าง") ไม่มีหน้าแยก

## Dual-Role ใน UI/UX

| สถานการณ์ | การแสดงผล |
|-----------|-----------|
| `/orders/[id]` ที่ฉันสร้าง | มุมมองผู้ซื้อ — ดูคนที่มาขาย, เลือกคนขาย, จ่ายเงิน |
| `/orders/[id]` ที่ฉันยื่นขายไป | มุมมองผู้ขาย — ดูขายตัวเอง, ดูไทม์ไลน์ |
| `/products` | ทุกคนเห็นเหมือนกัน (Entry Point หลัก) — มีทั้งปุ่ม ซื้อ/ขาย |

## Notification Event ที่เกี่ยวข้องกับ Watchlist

| Event | ลิงก์ไปที่ | เงื่อนไข |
|-------|-------------|----------|
| **มีงานซื้อใหม่ของสินค้าที่ติดตาม** | `/products/[id]` | ส่งทันทีเมื่อ PO ใหม่สถานะ `Open`, ไม่แจ้งเตือนคนสร้าง PO เอง |
