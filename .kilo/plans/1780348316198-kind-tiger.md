# Plan: สร้าง Front-end ทุกหน้าใหม่ — AgriForward

## หลักการ

- Mock data / local state เท่านั้น — ห้าม tRPC/Supabase ใน page component
- ภาษาไทย, คำศัพท์เกษตร ("งานซื้อ" ไม่ใช่ PO, "มาขาย" ไม่ใช่ Offer)
- Mobile-first, touch target ≥ 44px
- CSS var tokens เท่านั้น (ห้าม hardcoded color บน surface)
- Radius: `rounded-[var(--radius-2xl)]` card, `rounded-[var(--radius-xl)]` button/input, `rounded-[var(--radius-lg)]` list item, `rounded-full` circle/badge
- ใช้ `useToast()` แทน `alert()` ทั้งหมด
- ไม่ต้องใส่ comment ในโค้ด

## Phase 3: Orders — Flow ครบถ้วน

### 3.1 `/orders` — ปรับปรุงหน้า list

- เพิ่ม mock หลายสถานะ: Draft, Open, AwaitingSellerConfirm, PaymentPending, Contract, Fulfilled, Cancelled
- เพิ่ม status filter row (ทั้งหมด | เปิดรับ | รอชำระ | สำเร็จ | ยกเลิก)
- Buyer tab: แสดงงานซื้อ (PurchaseOrder)
- Seller tab: แสดงงานขาย (Offer) พร้อมสถานะที่ครบ (Open, Selected, Accepted, Withdrawn, Rejected)

### 3.2 `/orders/new` — ปรับปรุง flow สร้างงานซื้อ

- รับ `productId` จาก `useSearchParams()` และดึงข้อมูล product จาก mock
- ถ้าไม่มี productId → แสดง dropdown เลือกสินค้า (Select component)
- เพิ่ม delivery option selector (รับเอง/ส่งถึงที่) พร้อม address textarea
- เพิ่ม expiry date picker (งานซื้อหมดอายุเมื่อไร)
- สร้างสำเร็จ → toast + redirect `/orders/{newId}`
- ถ้า targetPrice ว่าง → tag "เปิดขายฟรี" (ให้เกษตรกรเสนอราคา)

### 3.3 `/orders/[id]` — ปรับปรุง detail + action buttons ตามสถานะ

**ตามสถานะ ต้องมี action:**
- Draft: ปุ่ม "เผยแพร่" (เปลี่ยน → Open) + "ลบ"
- Open: แสดง offers ที่มา + ปุ่ม "เลือกคนขาย" บน offer card
- AwaitingSellerConfirm: แสดง offer ที่ถูกเลือก + รอผู้ขายตอบรับ
- PaymentPending: ปุ่ม "จ่ายเงิน" sticky bottom (มีอยู่แล้ว)
- Contract/QC: แสดง QC result mock
- Fulfilled: ปุ่ม "ให้คะแนน" → `/ratings/new?orderId=...`
- Cancelled/Expired: แสดงเหตุผล

เพิ่ม:
- Mock หลายสถานะในหน้าเดียว (สลับดูได้ via temp state เพื่อ dev)
- ปุ่ม "เปิด dispute" (เมื่อ Contract หรือQC ไม่ผ่าน) → `/disputes/new?orderId=...`
- Comment thread มีอยู่แล้ว ✓

### 3.4 `/orders/[id]/checkout` — เพิ่ม address + payment method

- เพิ่ม delivery address form (ถ้าเลือก "ส่งถึงที่")
- เพิ่ม mock payment method selector (กระเป๋าเงิน AgriForward / โอนผ่านธนาคาร)
- แสดงยอดมัดจำ 5% (สูงสุด 1,000 บาท)
- สำเร็จ → toast "ชำระเงินสำเร็จ" + redirect `/orders/{id}`
